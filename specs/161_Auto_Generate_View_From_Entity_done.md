# Auto-Generate View from Entity

**Status**: ongoing

## User Value

> "As an enterprise architect, when I right-click any entity on the Architecture Canvas, I want to generate a new view containing that entity and everything connected to it -- so I can instantly get a focused, complete picture of an entity's ecosystem without manually adding dozens of related elements one by one."

## Problem

Creating a focused view for a specific entity requires manually creating a view and dragging in each related entity one by one. For a well-connected entity this can mean 20+ manual operations.

## Solution

**Purely frontend.** The canvas already loads all entities and their relations into memory. The relation DTOs contain both entity IDs (e.g., `ComponentRelationDTO` has `sourceComponentId`/`targetComponentId`, `RealizationDTO` has `capabilityId`/`componentId`, origin relationships have `componentId`/`acquiredEntityId` etc.). The frontend simply traverses these in-memory collections with a BFS — no new backend endpoint needed.

## Relation Types to Traverse

| Relation | Source ID Field | Target ID Field |
|----------|----------------|-----------------|
| Component Relation (Triggers/Serves) | `sourceComponentId` | `targetComponentId` |
| Capability Realization | `componentId` | `capabilityId` |
| Capability Parent | capability `id` | `parentId` |
| Origin: AcquiredVia | `componentId` | `acquiredEntityId` |
| Origin: PurchasedFrom | `componentId` | `vendorId` |
| Origin: BuiltBy | `componentId` | `internalTeamId` |

Capability Dependencies (Requires/Enables/Supports) are excluded — not currently rendered on the canvas.

## Traversal Algorithm

BFS over in-memory data the canvas already has:

1. Start with the selected entity in a queue, mark as visited
2. For each entity, find all directly related entities via any relation type (bidirectional)
3. Add unvisited entities to queue
4. Repeat until queue is empty or 500-entity safety cap reached
5. The visited set is the complete entity list for the new view

## Behavior

1. User right-clicks any entity node on the canvas
2. Context menu shows: **"Generate View for [entity name]"**
3. On click:
   a. Frontend traverses in-memory relations to collect all reachable entities
   b. Creates a new view named **"Autogenerated view for [entity name]"**
   c. Adds all discovered entities to the new view (using existing add-to-view commands)
   d. Switches to the new view
   e. Triggers auto-layout

## Edge Cases

| Case | Behavior |
|------|----------|
| Entity has no relations | View created with only the starting entity |
| Very large graph (500+) | Capped at 500, user informed via toast |
| Circular relations | Visited set prevents infinite traversal |
| View name exceeds 100 char limit | Truncate entity name in title to fit |
| User lacks view creation permission | Menu item not shown |

## Out of Scope

- Capability Dependencies (not on canvas yet — spec 039)
- Configurable traversal depth or relation type filtering
- Batch add-to-view API (use existing individual commands)
